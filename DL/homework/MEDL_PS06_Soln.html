Problem 1¶

In this problem, we would like to explore on how to decide the dimension of latent space in an autoenocoder. Let's use the MNIST data set and the autoencoder structure that we used in class.

  • Change the number of latent variables from 1 to 20 and
  • Compute the loss fuction for each case to see how well the autoencoder reconstructs the input
  • You are expected to plot the graph similar to the below one.

Then state your justification for the number of latent variables on your own words.

(Note that there is no right answer for the specific number of latent variable in this problem)

Hint: the basic idea or approach is similar to that of K-means).

import numpy as np
import matplotlib.pyplot as plt
import tensorflow as tf

mnist = tf.keras.datasets.mnist
(x_train, y_train), (x_test, y_test) = mnist.load_data()
x_train = x_train.reshape((x_train.shape[0],28*28)) / 255
x_test = x_test.reshape((x_test.shape[0],28*28)) / 255

train_imgs = x_train
train_labels = y_train
test_imgs = x_test
test_labels = y_test

loss_record = []
for i in range(19):

    encoder = tf.keras.models.Sequential([
        tf.keras.layers.Dense(500, activation = 'relu', input_shape = (28*28,)),
        tf.keras.layers.Dense(300, activation = 'relu'),
        tf.keras.layers.Dense(i + 2, activation = None)
        ])

    decoder = tf.keras.models.Sequential([
        tf.keras.layers.Dense(300, activation = 'relu', input_shape = (i + 2,)),
        tf.keras.layers.Dense(500, activation = 'relu'),
        tf.keras.layers.Dense(28*28, activation = None)
        ])
  
    autoencoder = tf.keras.models.Sequential([encoder, decoder])

    autoencoder.compile(optimizer = tf.keras.optimizers.Adam(0.001),
                        loss = 'mean_squared_error',
                        metrics = ['mse'])
    
    training = autoencoder.fit(train_imgs, train_imgs, batch_size = 50, epochs = 10)
    test_scores = autoencoder.evaluate(test_imgs, test_imgs, verbose = 2)
    
    loss_record.append(test_scores[1])
    
loss_record = np.array(loss_record)

plt.figure(figsize = (10, 10))
plt.plot(np.arange(len(loss_record)) + 2, loss_record, 'o-')
plt.show()

Problem 2: Conditional Autoencoder¶

Autoencoder (AE) is a neural network model which trains dimension reduction. The features shrunk by the encoder are suitable for reconstructing rather than valuable to solve our problem. Therefore, when we reconstruct a data from the feature on latent space we can find unwanted entanglement that hinders distinguish from class to class. However, the entanglement gives us some advantage as following. Discrete inputs produce discrete features on the latent space and the discrete features generate discrete outputs as well. But due to the entanglement, we can generate data from unseen points on the latent space so that the decoder become a model generates data from continuous distribution on the latent space. Thus, data generation is one of main purpose of AE and there are famous models such as variational autoencoder or adversarial autoencoder.



Conditional autoencoder (CAE) is one of the modified AEs that focus on data generation. It determines which class to generate in advance so that it can generate data of a specific class (or digit).

def cae_loss(y_true, y_pred):
    recon = tf.keras.backend.sum(tf.keras.backend.binary_crossentropy(y_true, y_pred), axis=-1) # what will be the reconstruction error?
    kl = 0.5 * tf.keras.backend.sum(tf.keras.backend.exp(l_sigma) + tf.keras.backend.square(mu) - 1. - l_sigma, axis=-1) #you don't have to worry about this part
    return recon + kl
